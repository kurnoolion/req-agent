# llm

**Purpose**
LLM abstraction layer. Defines the `LLMProvider` Protocol ‚Äî a single-method structural-typing contract (`complete(prompt, system, temperature, max_tokens) -> str`) that every other module uses to call an LLM. No module outside `src/llm/` imports an LLM SDK directly; new providers are added by implementing `complete()` on a class, with no base-class inheritance.

**Public surface**
- `LLMProvider` (base.py) ‚Äî `@runtime_checkable Protocol`; the only LLM interface the rest of the project sees
- `OllamaProvider` (ollama_provider.py) ‚Äî local Ollama HTTP client via stdlib `urllib`; `model`, `call_count`, `last_call_stats` (total_duration_s, eval_count, prompt_eval_count, tokens_per_second, model) for observability
- `MockLLMProvider` (mock_provider.py) ‚Äî deterministic keyword-based stub for tests and offline runs; `call_count`
- `HardwareInfo`, `ModelSpec`, `ModelChoice` (model_picker.py); `detect_hardware()`, `pick_model(hw, prefer=None)`, `list_available_ollama_models()`, `check_model_available()` ‚Äî auto-select the best Ollama model that fits detected RAM/VRAM

**Invariants**
- The `LLMProvider` Protocol is the single LLM seam. Changing its signature is a coordinated breaking change across every caller (taxonomy, query, eval).
- `complete()` returns plain text ‚Äî JSON parsing is the caller's responsibility. Keeps the Protocol minimal and portable to providers without structured-output modes.
- Default `temperature=0.0` ‚Äî all offline extraction paths want determinism.
- `OllamaProvider.__init__` pings `/api/tags` and raises `ConnectionError` on failure, and logs a warning if the requested model isn't pulled ‚Äî fail fast with a diagnosable message.
- `last_call_stats` is updated on every `complete()` call; callers snapshot it to record LLM category metrics (`LLM-*` error codes).

**Key choices**
- `Protocol` + structural typing over an ABC ‚Äî lets a vendor SDK wrapper satisfy the interface with zero inheritance, which matters when work-laptop constraints force swapping providers.
- Stdlib `urllib` for the Ollama HTTP call ‚Äî no `httpx` dependency, keeps the module installable on offline/locked-down hosts.
- Model selection is data-driven (`ModelSpec` list in `model_picker.py`), not wired into provider constructors ‚Äî hardware detection runs once and the pipeline config records the chosen model.

**Non-goals**
- Not a prompt library ‚Äî every caller owns its prompts; we don't centralize them here.
- No streaming, tool-use, or function-calling surface ‚Äî `complete()` is one-shot text in, text out. If a caller needs structured output, it parses JSON from the response.
- No retries, rate-limiting, or cost tracking at the Protocol layer ‚Äî providers implement whatever is needed for their own backend.
- No embedding surface ‚Äî embeddings live in [vectorstore](../vectorstore/MODULE.md) behind `EmbeddingProvider`.
