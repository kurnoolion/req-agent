# Telecom-requirement variant of SIRA's relevance_v04.txt.
# Same 0-100 scoring scale, retuned scoring guidance for normative
# device-requirement language.
Given a query and a device requirement, rate how useful the requirement is for answering the query.

Query: {query}

Requirement: {document}

Before scoring, follow these steps:
1. What is the query actually asking? Identify the specific information needed (a timer value, a procedure step, a feature applicability rule, a cause-code mapping, a spec citation, etc.).
2. Scan the requirement sentence by sentence: does ANY sentence contain or imply the answer? In MNO specs, the normative content typically uses "shall", "shall not", "should", or "may" — that's where the answer lives. Note that one sentence in the requirement with the answer is enough.
3. If the requirement is the section heading or anchor for the topic the query asks about (even with little body text), score at least 41 — heading-anchored requirements in MNO docs often delegate the actual behavior to nearby reqs, and finding the heading is the navigational first step.

Rate usefulness from 0 to 100:
- 0: completely unrelated topic, no shared specs / procedures / network elements / plan codes / feature names
- 1-20: peripherally related (shared spec family or RAT) but no sentence addresses what the query asks
- 21-40: discusses related concepts (same procedure family, same timer family, same applicability dimension) without the specific answer
- 41-60: the heading / anchor requirement for the queried topic, OR contains partial information (e.g. mentions the timer but not its value)
- 61-80: contains the answer, even if only in one normative sentence ("shall" / "shall not" / "should")
- 81-100: directly and clearly states the answer to the query, with the relevant normative verb

IMPORTANT: Score based on whether the ANSWER is present, not whether the requirement is "about" the query topic. A requirement that states "the UE shall set T3402 to 720 seconds" fully answers "what value does T3402 use" and scores 81+, even if the requirement also covers other timers. A requirement that lists "applicable form factors: smartphone, tablet" answers "which form factors does this feature apply to" and scores 81+.

IMPORTANT: A score of 0 means the requirement has NOTHING to do with the query — no shared 3GPP/GSMA spec, no shared procedure family, no shared network element, no shared plan code, no shared feature name. If the requirement mentions ANY entity from the query (a spec number, a plan code, a procedure name, a network-element name, a feature family), the score must be above 0.

Output only: {{"score": <integer>}}
